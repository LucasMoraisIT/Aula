Teste caixa-preta

Teste de caixa-preta é um teste de software para verificar a saída dos dados usando entradas de vários tipos. Tais entradas não são escolhidas conforme a estrutura do programa. Quanto mais entradas são fornecidas, mais rico será o teste.

Teste caixa-branca

Teste de caixa-branca é uma técnica de teste que usa a perspectiva interna do sistema para modelar os casos de teste. No teste de software, a perspectiva interna significa basicamente o código fonte. No teste de hardware, cada nó de um circuito pode ser testado.

Teste de tarefa

Os testes de tarefa são frequentemente usados em ambientes de trabalho para avaliar a aptidão de candidatos a determinados cargos, bem como para medir o desempenho de funcionários em seus papéis. Eles também são utilizados em pesquisas científicas para investigar habilidades cognitivas, comportamentais e técnicas em diversos contextos.

Teste comportamental

Os testes comportamentais geralmente consistem em uma série de perguntas ou declarações para as quais os participantes respondem indicando seu nível de concordância ou preferência. As respostas são então analisadas para identificar traços de personalidade, preferências de comunicação, estilos de liderança, aptidões interpessoais, entre outros aspectos comportamentais.

Teste intertarefas

Geralmente, esses testes são realizados em ambientes controlados, onde os participantes são fornecidos com um conjunto de problemas ou desafios de programação que devem ser resolvidos dentro de um determinado período de tempo. Os participantes normalmente usam uma linguagem de programação específica ou uma plataforma de desenvolvimento.

Teste de sistema

O teste de sistema é uma fase do processo de teste de software e de hardware em que o sistema já completamente integrado é verificado quanto a seus requisitos num ambiente de produção. Está no escopo da técnica de teste de caixa-preta, e dessa forma não requer conhecimento da estrutura interna do sistema.
